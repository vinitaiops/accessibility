Good Example: Explicit Labels

These form inputs have labels explicitly associated with them. Explicit labels make the association unambiguous for assistive technologies. Note: This is the recommended method for most circumstances..

                            <form>
                            <p>
                                <label for="fname_a">First Name:</label> 
                                <input type="text" name="fname_a" id="fname_a">
                              </p>
                              <p>
                                <label for="lname_a">Last Name:</label>
                                <input type="text" name="lname_a" id="lname_a">
                              </p>
    
                          </form>

Good Example: Implicit Labels

An implicit association is created by putting the form control inside the label element.

                          <form>
                          <p>
                               <label>First Name: <input type="text" name="fname1"></label>
                           </p>
                           <p>
                              <label>Last Name: <input type="text" name="lname1"></label>
                           </p>
                     </form>

Good Example: aria-label for a text input

This example shows a search input field that uses aria-label to provide a label for screen readers, since there is no visible label. The search field is visibly labelled by the search button next to it. This is one of the few circumstances in which it is permissible to not have a visible label for a form field.

                          <form>
                            <p><input type="text" aria-label="search"> <input type="submit" value="Search"></p>
                         </form>

Good Example: title attribute for a text input

In this example, the title attribute is used as the screen-reader accessible label for the text field. The title attribute is invisible to sighted users, though it does cause a tooltip to appear when users hover the mouse over the text input, which can be considered an accessibility benefit. Note that the tooltip does not appear when the user focuses on the input with the keyboard, so the usefulness of the tooltip of the title attribute is limited only to mouse users.

                          <form>
                              <p><input type="text" title="search"> <input type="submit" value="Search"></p>
                          </form>
